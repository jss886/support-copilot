### CLI 入口概述
当前 `cli` 目录里的命令行能力由 `cli/args.py` 定义参数、`cli/handlers.py` 执行具体逻辑、`cli/main.py` 作为入口调用。整体上这是一个围绕 RAG 知识入库、检索和问答的命令行工具。

### index
把单个本地文档切分成多个文本块，生成向量后写入 PostgreSQL。适合导入一份指定文件作为知识库数据源，主要参数是 `--source`，并支持配置分块大小、分块重叠、批量写入大小和数据库连接信息。

### index-dir
批量读取一个目录下的 Markdown 文件，切分后统一写入 PostgreSQL。适合一次性导入一批本地知识文档，主要参数是 `--source-dir`，同时支持分块和数据库相关配置。

### index-feishu
读取单个飞书 Docx 文档，切分并写入 PostgreSQL。适合把飞书中的某篇文档直接纳入知识库，主要参数是 `--doc-id`，其余参数用于控制分块和数据库写入。

### index-feishu-db
读取单个飞书 Docx 文档并写入 PostgreSQL，实际执行逻辑和 `index-feishu` 一样，也是调用同一个入库函数。代码层面看，它更像是一个命名上单独保留出来的飞书入库命令。

### index-feishu-space
递归读取飞书知识库某个空间节点下的 Docx 文档，并批量写入 PostgreSQL。适合整批同步飞书知识库内容，
主要参数是 `--space-id` 和 `--parent-node-token`。

### seed-test-data
生成模拟的客服/支持类测试文档，并把切分后的数据写入 PostgreSQL。
适合本地演示、联调或构造基础测试数据，主要参数是 `--doc-count` 和 `--chunks-per-doc`。

### seed-hard-negatives
生成“看起来相关但实际错误”的干扰文档并写入 PostgreSQL。这个功能主要用于构造更难的检索测试集，
方便验证召回和排序在复杂场景下的表现。

### query
根据用户问题去 PostgreSQL 中检索最相关的文本块，并把命中的片段直接打印出来。
主要参数是 `--question` 和 `--top-k`，还支持通过 `--source` 按来源过滤，适合单独验证召回效果。

### answer
先检索相关文本块，再基于检索结果生成最终答案并输出。
主要参数同样是 `--question`、`--top-k` 和可选的 `--source`，适合直接验证完整 RAG 问答链路。

### 通用参数能力
大多数命令都支持传入 `--jdbc-url`、`--db-user`、`--db-password` 来指定 PostgreSQL 连接信息，
同时支持 `--embedding-dimensions` 配置向量维度。
涉及入库的命令还普遍支持 `--chunk-size`、`--chunk-overlap`、`--batch-size` 来控制切分和写入策略。
