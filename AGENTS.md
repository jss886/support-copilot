# Skill 名称
Chinese Commenting Style

# 目标
在编写、修改、重构代码时，始终保证注释清晰、简洁、中文化，方便后续阅读和维护。

# 核心要求

## 1. 函数注释要求
每次新增或修改函数时，必须在函数上方补充中文注释，至少说明：
- 这个函数的主要作用
- 必要时补充输入 / 输出 / 关键约束

要求：
- 注释使用中文
- 语言简洁自然，不要写成流水账
- 不要只重复函数名，要说明“这个函数是干什么的”

示例：
# 作用：根据用户问题从知识库中召回最相关的文档片段
def retrieve_docs(query: str) -> list[str]:
    ...

## 2. 复杂逻辑注释要求
在以下位置，需要补充简要中文注释：
- 分支逻辑比较绕的地方
- 容易误解的边界条件
- 非直观的数据转换
- 魔法值 / 特殊规则
- 与外部系统交互时的重要约束
- 性能优化、兼容性处理、兜底逻辑

要求：
- 注释只解释关键点
- 优先解释“为什么这样做”，而不是逐字翻译代码
- 一般控制在 1 行，必要时 2 行
- 不要对非常直白的代码加无意义注释

示例：
# 这里保留前 5 条结果，避免低质量长尾结果影响后续 rerank
top_candidates = candidates[:5]

## 3. 注释语言要求
所有新增注释、修改后的注释、补充的说明，都必须使用中文。

要求：
- 中文表达自然
- 不要中英混杂，除非是专有名词、接口名、类名、字段名
- 若原有英文注释对应代码被修改，优先同步改成中文

## 4. 注释质量要求
注释应满足以下原则：
- 简洁：不写大段空话
- 有用：帮助理解代码意图
- 准确：注释与代码逻辑保持一致
- 克制：不对显而易见的代码逐行解释

避免这种低质量注释：
# 定义一个变量 x
x = 1

更推荐这种：
# 这里用 set 去重，避免重复 chunk 进入召回结果
unique_chunks = list(set(chunks))

## 5. 修改代码时的要求
当你修改已有代码时：
- 如果函数逻辑变了，要同步更新函数注释
- 如果复杂逻辑变了，要同步更新对应注释
- 如果旧注释已经不准确，要直接修正，不要保留过时注释

## 6. 默认执行策略
除非用户明确要求不要加注释，否则你在生成代码时默认执行以下策略：
- 每个函数都写中文作用注释
- 每个复杂点都补必要的中文简要注释
- 保持整体代码整洁，不要因为注释过多影响可读性

# 输出风格
你输出的代码应满足：
- 注释全部为中文
- 函数上方有作用说明
- 难点处有简要注释
- 注释风格统一、朴素、工程化
- 不要写得像文档说明书，保持开发者日常代码风格
