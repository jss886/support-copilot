# Multi-Agent Support Copilot 项目概览

## 项目目标

本项目目标是构建一个面向企业内部支持场景的 Copilot。系统需要接入飞书文档、知识库、FAQ、API 说明和历史故障案例，并通过 RAG 提供基于知识的问答能力。

除了普通问答，本项目还希望在每次对话结束后提炼出可以复用的经验，并将这些经验沉淀到 `task_memory`，形成可持续增长的知识资产。

## 典型使用场景

1. 用户询问某个系统报错应该如何排查。
2. 用户想确认某个接口的请求参数和返回字段。
3. 用户咨询某个模块依赖关系，想知道问题可能由哪个上游系统引起。
4. 用户连续追问同一个问题，希望系统保留上下文并给出更完整的建议。

## 核心能力

### 1. 文档检索

系统接入飞书文档、项目说明、接口手册和排障文档后，会对文档做切片、向量化，并存入 PostgreSQL 与 pgvector。检索时同时结合关键词召回和语义召回。

### 2. 多 Agent 协同

系统内部可按任务类型拆分多个 Agent，例如：

- routing_agent：识别问题类型并决定处理路径
- retrieval_agent：执行知识检索
- action_agent：处理工具调用
- answer_agent：基于上下文生成最终回答
- memory_agent：从本轮会话中提取经验并写入记忆

### 3. 经验沉淀

如果一轮问答中出现了明确结论、可复用排查路径、典型故障经验或可抽象规则，系统应将其整理为结构化结论并写入 `task_memory`。

## RAG 的基本流程

1. 读取文档内容。
2. 按固定大小进行切片，并设置一定重叠，避免语义被硬切断。
3. 调用 embedding 模型生成向量。
4. 文本正文存入 PostgreSQL，向量写入 pgvector。
5. 查询时同时执行语义检索与关键词检索。
6. 对检索结果做融合和重排。
7. 将最终上下文交给问答模型生成回答。

## 当前基础版本

当前仓库中的 `basic_rag.py` 是一个最小可运行示例。它先对单个文档切片，再调用阿里百炼兼容接口生成 embedding，并将结果保存到本地 JSON 文件中。这个版本还没有真正接入数据库，但可以先用来验证切片、向量化和检索链路是否跑通。
