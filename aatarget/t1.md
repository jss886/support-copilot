## 今天目标
1. 实现一个最基础的rag 进行一个文档的切片 然后调用阿里的 text-embedding-v4 模型进行一个embedding 
2. 然后进行一个根据用户问题进行一个rag检索
3. 对接飞书

### 关键难点
1. 飞书文档里面包含的图片怎么解析  
2. 使用什么Loader

### 解决方案
1. 图片的话用OCR解析 也就是提取图片中的文字拼接到文档里面 暂时不考虑用多模态大模型去进行一个操作 代价太大了
2. Loader使用自定义的FeishuLoader 因为飞书文档本就是结构化的 由各个block组成的文档 无需用UnstructuredLoader
其次就是自定义的Loader能更好的保存飞书文档的元信息 比如document_id、block_id、标题层级知识库节点路径、文档来源链接等

### 对接飞书具体操作
https://open.feishu.cn/app/cli_a94535edde39dccc/version/7625149313065225419

在上面的飞书开放后台创建一个应用 -> 给其对应的权限 -> [此时就可以通过app_id与app_secret进行文档的读取和写入了] 
-> 添加机器人插件 -> 在飞书app里面新建群组 -> 群组设置里面找到机器人选项将机器人添加进来 
-> 知识库设置里面添加管理员 搜索群聊名称 -> 将对应群聊添加为管理员 -> [此时就可以通过api获取]
